All About GitHub

  • Q. What is a version control system
  • A. Tracking of what changes in code
  • Q. What is a Git?
  • A.Record of programming versions
  • Q. What is GitHub?
  • A.A more social form of a Git or version control system
  • Q. What is the difference between Git and GitHub?
  • A.A git does not require internet connnection and is locally sourced. GitHub requires an internet connection to post your programs and test others.
  • Q. Who started GitHub and how was it started?
  • A.GitHub was launched by Tom Preston-Werner, Chris Wanstrath, P.J. Hyett and Scott Chacon in October 19th of 2007. It was started after the previous popular Version Control system Bitkeeper came with a price tag.
  • Q. What company owns it now?
  • A.Microsoft Corporation
  • Q. How much does GitHub account cost?
  • A.Its free!
  • Q. What is the Octocat?
  • A.GitHubs Mascot
  • Git/GitHub terms: Repository:Contains all project files and their revisions. Commit:Execute or publish your code. Fork:A copy of a repository one owns. Push:Updates a remote branch with local commits. Pull Requests:Allows one to inform others of changes made to a branch in a repository. Workflows:Automated process that runs jobs. Issues:Lets you track ones work in GitHub. Raw Button:Allows access to view raw file without styling. Blame Button:Allows user to see each line of code with a timestamp and what user made those changes.
