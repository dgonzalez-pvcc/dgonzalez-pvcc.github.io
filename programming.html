Phases of Computer program development

  • Phase 1: Understand the problem
  • Phase 2: Design an algorithm to solve the problem
  • Phase 3: Write the program code in appropriate language based on the algorithm
  • Phase 4: Enter the program and save
  • Phase 5: Execute the program
  • Phase 6: Test the program, find errors and debug, then retest
  • Phase 7: Document and evaluate the program

    • Programming Structures

    • Sequential structure: Top-down execution
    • Decision structures: if/then/else
    • Iterative (Looping): while looping: continue looping as long as something is true repeat/until loop: continue looping until something is true for loop: loop a given number of times
    • Recursive structures: Repeat a set of instructions as a subtask of itself
    • Repitition continues until: degenerative case, which is a termination condition, is met (degenerative case is also called a "base case")

    Two classic programming/database problems

    • Sorting numbers
    • Sorting strings (in accordance with ASCII values)
    • Searching: find specific data that meet a certain criteria
    • In database: Filters: temporary search Queries: search criteria are saved so they dont have to be recreated
    • Two popular search techniques: Sequential search: search from top to bottom or bottom to top, one item at a time Binary search: only works if data is SORTED first, start in the middle, then go to half of that, etc.

    Two classic program goals

    • Effiiciency: Program takes minimal time and/or resources Correctness: Program runs in expected ways and works for all expected conditions

    Software Testing and Verification

    • Kinds of tests:
    • Acceptance Testing: Verifying whether the whole system works as intended.
    • Integration Testing: Ensures that software components or funcitons operate together.
    • Unit Testing: Confirms that software unit performs to expectations. A unit being the smallest testable component of an application.
    • Functional Testing: Checks functions by emulating scenarios, based on functional requirements. Black-box testing being a common way to verify functions
    • Performance Testing: Tests how the software performs udner different workloads.
    • Regression Testing: Checks whether new featueres break or degrade functionality.
    • Stress Testing: Tests the strain a system can take before it fails.
    • Usability Testing: Validates how easily a user can operate the system or web application.
      • Software Testing is done to detect:
      • Architectual flaws
      • Poor Design Deicisions
      • Invalid or incorrect functionality
      • Security vulnerabilities
      • Scalability issues
      Testing best practices:
      • Continous Testing: During the entirety of product lifespan development and operations collaborate testing the product as opposed to right before it is provided for mass consumption.
      • Configuration Management: Organizations maintain test assests and track what software builds to test.
      • Servic Virtualization: Correct testing environemnt is not available yet so servives and systems are emulated.
      • Defect or "Bug Tracking": Monitors defects before testing.
      • Metrics and Reporting: This allows the working team to keep a record of changes to goals, test results and more important data points.
